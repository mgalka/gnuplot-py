Gnuplot.py

Gnuplot.py is a Python module that interfaces to gnuplot, the popular plotting program. It allows you to use gnuplot from within Python to plot arrays of data from memory, data from files, or mathematical functions. If you use Python to perform computations or as `glue' for numerical programs, you can use this module to plot data on the fly as they are computed. And the combination with Python makes it is easy to create crude `animations' by plotting different datasets one after another; the datasets can even be written to temporary files in advance of the animation to increase the frame rate.

Communication with gnuplot is through a pipe (for commands) and temporary files (for data). It has been written and tested on a unix computer; I don't know whether it will run under other operating systems (please let me know if you try).

This module has an object-oriented design that allows the user flexibility to set plot options and to run multiple gnuplot sessions simultaneously. If you are more ambitious, it would not be difficult to program new types of plottable items by deriving from the `PlotItem' class. For a demonstration, run the python file by typing `python Gnuplot.py'.

News

Version 1.1 has been released (31 Jan 1999). New features:
  • 3-D plots are now supported through the new `splot' method:
    • To create a scatter plot, pass splot 2-D an array containing (x,y,z) triplets;
    • To create a grid plot, pass a 3-D array consisting of a 2-D array of (x,y,z) triplets; or
    • Use the new `GridData' PlotItem to plot gridded data (data tabulated on a rectangular grid).
  • It is now easier to pass data to gnuplot. The `Data' PlotItem now accepts multiple arguments representing subsequent `columns' of the dataset; e.g., what used to be written as 
              g = Gnuplot.Gnuplot()
          x = Numeric.arange(100)/10.0
          y = x**2
          # Create an array of (x,y) pairs:
          g.plot(Gnuplot.Data(Numeric.transpose((x, y))))
    can now be shortened to 
              # Plot y vs. x directly:
          g.plot(Gnuplot.Data(x, y))
    (both examples plot x squared vs. x). Similarly, splot can be passed three arrays representing the values of x, y, and z. Note that the old interpretation is still used if only a single argument is passed to `Data'.

Documentation

There is still no separate documentation for this package, but the Gnuplot.py file contains a huge number of documentation strings that you can read by opening the file in an editor. There is also a demonstration that you can run by typing `python Gnuplot.py', then look at the bottom of the Gnuplot.py file to see how the demonstration works. To get good use out of Gnuplot.py, you will want to know something about gnuplot, for which a good source is the gnuplot help (run gnuplot then type `help', or read it online at this website).

Installation

Before installing Gnuplot.py, first make sure that you have working versions of
  • the gnuplot program,
  • Python, and
  • the Numeric Python extension
To install Gnuplot.py:
  1. Download Gnuplot-1.1.tar.gz.
  2. Gunzip and untar it, which will create a directory called ``Gnuplot-1.1''. (For now, the directory only contains one file, ``Gnuplot.py''.)
  3. To test the package, change to that directory and type ``python Gnuplot.py''.
  4. Copy Gnuplot.py into a directory that is in your PYTHONPATH, or add the ``Gnuplot-1.1'' directory to your PYTHONPATH.
If you don't have gunzip or tar, you can download Gnuplot.py directly from here.
