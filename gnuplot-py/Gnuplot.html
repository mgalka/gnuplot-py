Intro

Gnuplot.py is a Python package that interfaces to gnuplot, the popular plotting program. It allows you to use gnuplot from within Python to plot arrays of data from memory, data files, or mathematical functions. If you use Python to perform computations or as `glue' for numerical programs, you can use this package to plot data on the fly as they are computed. And the combination with Python makes it is easy to automate things, including to create crude `animations' by plotting different datasets one after another.

Commands are communicated to gnuplot through a pipe and data either through the same pipe (as "inline" data) or through temporary files. It has been written and tested on a unix computer but is also reported to work under MS Windows, Macintosh, and possibly other operating systems (please give me feedback!)

This package has an object-oriented design that allows the user flexibility to set plot options and to run multiple gnuplot sessions simultaneously. If you are more ambitious, it is not difficult to add entirely new types of plottable items by deriving from the `PlotItem' class.

For a demonstration, run the python file by typing `python demo.py'.

News

Version 1.4 has been released (11 Oct 1999). This version includes the following changes:
  • Added support for the Macintosh.
  • Separated the platform-dependent code into separate modules: gp.py, gp_mac.py, and gp_win32.py.
  • Allow access to all of the postscript printer driver options through the hardcopy() command.
  • Fixed an import problem in test.py.

Version 1.3 (released 23 Sep 1999) introduced the following features:

  • Converted to package format. The main file is now called __init__.py, so that it can be loaded by typing `import Gnuplot'.
  • Passing a callable function to GridData was broken; that functionality is now provided by a new type of PlotItem called `GridFunc'.
  • Added a new keyword argument, `ufunc', to grid_function and to GridFunc, to request that the function be evaluated matrix-wise.
  • Separated the demonstration code into a separate file, demo.py, and added a test.py module to test the features of Gnuplot.py.
To read about changes introduced in earlier versions, read the NEWS file.

Documentation

The quickest way to learn how to use Gnuplot.py is to run the simple demonstration by typing `python demo.py', then to look at the demo.py file to see the commands that create the demo. One of the examples is probably similar to what you want to do.

There is also online documentation for Gnuplot.py, extracted automatically from the package's docstrings using pythondoc. Or you can download the html documentation to your machine in tar or zip format.

Finally, you can look at the docstrings yourself by opening the file __init__.py in an editor.

To get good use out of Gnuplot.py, you will want to know something about gnuplot, for which a good source is the gnuplot help (run gnuplot then type `help', or read it online at this website). You might also want to check out Bernhard Reiter's gnuplot information page, which has many more links.

Installation

More detailed installation instructions are contained in the README file. Before you can use Gnuplot.py, you will need working versions of
  • the gnuplot program,
  • Python (the version should be at least 1.5), and
  • the Numeric Python extension
If you want to run under MS Windows, make sure you have the Win32 extensions installed. Also make sure you have pgnuplot.exe (see the README file for details).

To install Gnuplot.py:

  1. Download either Gnuplot-1.4.tar.gz or Gnuplot-1.4.zip.
  2. Gunzip and untar (or unzip) it, which will create a directory called ``Gnuplot-1.4''.
  3. Refer to the README file in that directory for further instructions.
